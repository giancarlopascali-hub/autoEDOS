Running the Optimization

A step-by-step walkthrough of the autoEDOS workflow — from your first experiment to a fully optimized result.

The Big Picture

autoEDOS follows a closed-loop optimization strategy. Instead of testing combinations randomly (OVAT), the platform learns from every experiment and intelligently narrows in on the best conditions.

1

Plan

Design a small, information-rich set of initial experiments using DoE

↓

Run experiments in the lab, fill in the results

2

Optimize

Upload results, run BO, get new conditions. Repeat until goals are met.

3

Analyse

Review trends, feature impacts, and build predictive models in SA

Shortcut: Already have data? Skip straight to Step 2 (BO). You don't need to start from DoE.

Step 1 — Plan Your Experiments (DoE) OPTIONAL

Use the DoE module when starting from zero. It generates a mathematically optimal set of initial experiments that maximally cover your parameter space with the fewest runs.

  1. Define your input variables (Features) Go to the DoE module → Features tab. Add each parameter you will vary (e.g. Temperature, Catalyst). Set the type (Continuous, Discrete, Categorical) and its range.
  2. Name your output variables (Objectives) In the Objectives tab, add the metrics you will measure (e.g. Yield, Purity). You don't set targets here — just names.
  3. Choose a design model and generate Pick a DoE model in Tweaks (LHS is safe for any number of variables). Click Create table of experiments. Review the quality metrics, then export the CSV.
  4. Run the experiments and fill in results Perform the experiments in the lab. Open the exported CSV in Excel and fill in the measured values for each objective column. Save the file.
What the quality metrics mean: Orthogonality — how independent your features are (higher is better). D-Efficiency — overall coverage of the parameter space. Resolution — ability to distinguish interaction effects. Curvature — whether the design can detect non-linear responses.

Step 2 — Bayesian Optimization (BO) CORE LOOP

This is the engine of autoEDOS. After each lab campaign, upload your results and BO will propose the next best set of conditions to try. Repeat this cycle until your goals are met.

  1. Upload your results CSV Switch to the BO module. Drag and drop your CSV (with results filled in) into the upload zone. The app will automatically detect your columns.
    Cold Start: No data yet? You can upload a CSV that contains only the column header row (feature and objective names, no data rows). BO will then generate a smart set of initial conditions using space-filling sampling — no prior experiments needed.
  2. Assign column roles (F / O) In the data table header, each column has an F/O toggle. Set your input variables to F (Feature) and your measured outputs to O (Objective).
  3. Configure Features In the Features tab, set the type and range for each feature. Continuous means the AI can suggest any value in the range. Discrete/Categorical restricts suggestions to specific values.
  4. Configure Objectives In the Objectives tab, tell the optimizer what to do with each output: Maximize, Minimize, or hit a specific Target value. For multi-objective problems, set the relative importance of each goal using the percentage inputs.
  5. Run the optimizer Click Run Bayesian Optimization. The AI fits a surrogate model (Gaussian Process) to your data, then searches for the conditions most likely to improve your objectives. Suggestions appear in a green-highlighted table.
  6. Fill in results and commit Run the suggested experiments in the lab. Type the measured values directly into the green cells. Once all values are entered, click Add all to Main Dataset. The data is merged and all plots refresh.
  7. Repeat until done Go back to Step 1 of this loop — upload the updated dataset (or just export it from the app). Each cycle, the model improves its understanding and narrows in on the optimum. Typically 3–6 cycles is sufficient.
How BO actually works: After each experiment, the Gaussian Process model updates its probabilistic map of the entire parameter space. It balances exploitation (testing near known good results) and exploration (probing uncertain regions). This is far more efficient than grid search or random testing.

Step 3 — Statistical Analysis (SA) INSIGHTS

Once you have enough data (typically 15+ experiments), use the SA module to understand why the optimum exists — not just where it is.

  1. Load data and select variables Upload (or keep loaded) your full dataset. In the SA module, check which features and objectives to include in the analysis.
  2. Choose a model and run In SA Tweaks, select a model. Linear Regression for simple, interpretable trends. Neural Network/MLP for complex non-linear interactions. Click Run the Statistical Analysis.
  3. Interpret the results Review the correlation matrix, SHAP importance plots, and categorical interaction charts. Use the Estimator sliders to simulate what-if scenarios and predict outcomes for any combination of conditions.
  4. Export your report Click Export Analysis Report to download a fully interactive HTML dashboard that works offline — ideal for sharing with collaborators or adding to publications.

Automation Mode ADVANCED

What is autoEDOS-IMEX?

autoEDOS-IMEX (IMport EXport) is a companion software that runs on the local laboratory computer and acts as a bridge between autoEDOS and your physical instruments. It handles all data exchange exclusively in the form of .csv files, supporting the following communication channels:

  • Local folders: Watches a designated folder for incoming result files and writes proposal files for the instrument software to pick up.
  • Serial / COM port: Communicates directly with instruments connected via RS-232 or USB-serial adapters.
  • Network (TCP/UDP): Sends and receives CSV data over a local network or internet connection — useful for remote instruments or cloud-connected platforms.
  • USB: Direct USB data exchange with compatible instruments.

In short: IMEX is what makes the loop fully closed. Without it, you manage experiment dispatch and result upload manually. With it, the entire DoE → Run → Optimize cycle runs hands-free.

Path A — Starting from DoE

Use this path when you have no prior data and want to generate a statistically designed initial experiment set automatically.

  1. Start autoEDOS-IMEX & confirm (Step 1) Run the IMEX software on your lab computer. Check the confirmation box in Step 1 of the Auto module.
  2. Select "Start from DoE" (Step 2) Click Start from DoE. You will be taken to the DoE module. Define your features, objectives, and design model, then click Create table of experiments.
  3. Return to Auto and perform the Auto DoE Click Back to Auto Setup. A button labeled Perform Auto DoE will appear. Clicking it sends the DoE experiment table to autoEDOS-IMEX, which dispatches the experiments to your instrument. The app waits for the results to return automatically.
  4. Results received — configure BO (Step 3) Once IMEX delivers the experimental results, they are automatically loaded into the dataset. You are prompted to check the BO setup. Click Check BO Setup to go to the BO module and confirm your objectives, features, and tweaks are correct.
  5. Set stopping conditions (Step 4) & launch Return to Auto. Set your stopping condition (iterations, no improvement, target, or time), then click Start Automated BO. The BO loop runs fully autonomously from this point.

Path B — Starting from BO

Use this path when you already have experimental data (e.g. from a previous DoE) and want to go directly into the optimization cycle.

  1. Start autoEDOS-IMEX & confirm (Step 1) Run the IMEX software on your lab computer. Check the confirmation box in Step 1.
  2. Select "Start from BO" (Step 2) Click Start from BO. You will be taken to the BO module. Upload your existing data CSV and configure all features, objectives, and tweaks.
  3. Confirm BO setup (Step 3) Click Back to Auto Setup, then Check BO Setup to mark this step complete.
  4. Set stopping conditions (Step 4) & launch Set your preferred stopping condition, then click Start Automated BO. Each cycle, IMEX receives the new proposals, runs them on the instrument, and returns results to trigger the next iteration — all automatically.
Stopping Conditions:
  • Number of iterations: Stop after N complete BO cycles.
  • Best conditions not improved for X iterations: Stop if the best result found hasn't improved over the last X cycles — useful for convergence detection.
  • % Target reached: Only active when all objectives are set to "target" mode. Stops when the target threshold is met.
  • Time: Stop after a maximum elapsed time (hh:mm).

1. Design of Experiments (DoE) PLANNING

Use this module when starting from scratch to create a statistically sound plan for your first experiments.

Feature Types

  • Continuous: Any value in a range. Format: [min, max]
  • Regular: Fixed-step grid. Format: [min, max]N (e.g. [20, 100]5 = 5 equally spaced points)
  • Discrete: Specific numeric values. Format: 10, 25, 50, 100
  • Categorical: Named options. Format: Catalyst A, Catalyst B, Catalyst C

DoE Models

  • Latin Hypercube Sampling (LHS): Space-filling design. Works with any number of factors. Best for flexibility and maximum coverage with a free choice of N experiments.
  • Box-Behnken: Efficient for 3–7 factors. Excellent for quadratic surface modelling.
  • Definitive Screening (DSP): Very efficient for many factors. Can detect main effects and some interactions.
  • Full Factorial: Tests every combination. Only practical for few factors with few levels.
  • Plackett-Burman: Screening design. Good for identifying the most important factors quickly.

Quality Metrics

  • Orthogonality: How independent the features are from each other (closer to 100% is better).
  • D-Efficiency: Overall statistical efficiency of the space coverage.
  • Resolution: Indicates which interaction and quadratic effects can be estimated without confounding.
  • Curvature: Whether the design permits modelling of non-linear responses.

2. Bayesian Optimization (BO) OPTIMIZATION

The "brain" of the app. It analyzes your current data and recommends the next best settings to reach your goals.

Cold Start: You don't need prior data to use the BO module. Upload a CSV that contains only the header row (column names separated by commas, no data). The optimizer will generate a smart initial set of experiments using Sobol space-filling sampling — a statistically superior alternative to random guessing.

Objective Modes

  • Maximize: Push the objective as high as possible (e.g. yield, conversion).
  • Minimize: Push the objective as low as possible (e.g. cost, byproduct).
  • Target: Hit a specific value (e.g. pH = 7.0, purity = 98%). The optimizer penalizes deviations in either direction.

Multi-Objective Strategy

  • Discovery Mode (equal importance): Uses NEHVI to explore the full Pareto front — all trade-off solutions between objectives.
  • Priority Mode (unequal importance): Uses Scalarized EI to strictly prioritize the high-importance objectives.

BO Tweaks

  • Batch Size: How many new experiments to propose per cycle.
  • Optimization Strategy: Gradient-based (default, precise for continuous) or Exhaustive Grid (tests all defined combinations, ideal for fully discrete spaces).
  • Acquisition Function: LogEI (confident improvements), qUCB (balanced exploration).
  • Kernel: Mathematical assumption about the response surface. Matern 5/2 is standard. RBF is smoother.
  • Exploration vs. Exploitation slider: Controls risk appetite. Left = exploit known good areas. Right = explore unknown regions.
  • Noiseless: Enable only if your measurements have zero variance/error.
  • Avoid Re-evaluations: Prevents suggesting conditions already in your dataset.

The Pareto Front

The Pareto chart shows experiments that achieve the best balance of all objectives simultaneously (orange markers). These are the "non-dominated" solutions — no other experiment beats them on all objectives at once. The chart always displays raw, unweighted values to keep visualisation grounded in physical reality.

3. Statistical Analysis (SA) INSIGHTS

Go beyond optimization to understand why your experiments behave the way they do.

Models

  • Linear Regression: Fast, interpretable. Use for simple trends and few interactions.
  • Neural Network/MLP: Captures complex non-linear interactions. Requires more data (20+ points).

Understanding the Results

  • Correlation Matrix: Heatmap of how strongly each numeric feature affects your goals.
  • Categorical Interactions: Shows how combinations of categorical levels impact the success score.
  • Feature Impact (SHAP): A dual-panel plot.
    • Left panel (% Impact): Relative importance of each variable across the whole dataset.
    • Right panel (SHAP Beeswarm): Dots pushed right = increased the objective. Dots pushed left = decreased it. Red = high parameter value, Blue = low.
  • Estimator Tool: Sliders for predicting what-if scenarios. Shows predicted value and a Success Score.

4. Data Management TIPS

CSV Format

  • Standard CSV: Columns separated by commas. The first row must be the header (column names).
  • Column naming: Use descriptive names. Avoid special characters.
  • Missing values: Leave objective cells empty or use "N/D" for experiments not yet run. The optimizer ignores these rows during training but preserves them in the dataset.
  • Cold start CSV: A CSV with only the header row (no data) is valid. The BO optimizer will use it to generate an initial set of experiments from scratch.

In-App Editing

Click any cell in the main data table to edit its value. Click away to save. Useful for correcting typos or entering results from the lab directly into the app.

Column Role Assignment

Use the F/O toggle in each column header to designate columns as Features (inputs) or Objectives (outputs). This assignment is remembered across all modules.

Data Privacy

All data processing happens locally on your machine or within your private Hugging Face Space. Your experimental data is never shared with third parties or used to train global models.
